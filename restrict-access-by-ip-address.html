Restrict access to Uptempo by IP address

This feature is available in Preview with the 2026.2 release. To enable this feature in your Uptempo instance, please contact your Customer Success Manager, or Uptempo Support.

By default, an Uptempo instance is not IP address-restricted: any user with valid credentials to the instance can successfully sign in from any network.

If your organization requires additional security, you can configure an IP allowlist to specify which IP addresses are allowed to connect to your Uptempo environment. For example, you can require that users can only sign in to your Uptempo instance if they are on an office network, or are connected to your VPN.

When you configure an IP allowlist, Uptempo will only accept login attempts made from IP addresses that are on the allowlist. This additional layer of security helps to prevent unauthorized access in the event of a password leak, as login attempts from unrecognized IP addresses are automatically blocked — even if the credentials are still valid.

How the IP allowlist works

To configure the IP allowlist, you add approved IP addresses to the list. The allowlist is automatically enabled after at least one IP address has been added.

When the allowlist is enabled, the authentication system checks every login attempt against the allowlist. If the attempt is using valid login credentials, and is coming from an IP address that is:

  • On the allowlist: The login is approved as usual.

  • Not on the allowlist: The login is rejected, and an error is displayed:

Supported authentication methods

IP allowlist functionality is supported for these authentication methods:

  • Web authentication (email address and password)

  • SAML single sign-on (SSO) authentication

The IP allowlist functionality currently does not apply to authentication to the Uptempo API.

Before you begin

Using this feature requires:

  • An Uptempo 2.0 instance

  • The Optimize package

Before you start configuring the IP allowlist, obtain:

  • Your own IP address (you must add this first, to avoid locking yourself out)

  • All IP addresses that must be added to the allowlist
