Configure Estimated Costs options

As an administrator, you can customize the Estimated Costs feature to your organization's needs. You can choose:

  • What the default date range for estimated costs should be when an activity does not have in-market dates set

  • Whether users enter estimated costs as quarterly or total amounts.

Set the default date range and input format for Estimated Costs

  1. In the Activities section, click Settings:

  2. In the Activity Configuration menu, click Measures > Estimated Costs.

  3. Use the Default Date Range menu to select the default date range you want to use for estimated costs. Choose between the following options:

    This Year

    Sets the default date range to the current calendar or fiscal year.

    Next Year

    Sets the default date range to the next calendar or fiscal year.

  4. Use the Time Granularity menu to select the granularity with which users enter estimated costs. Choose between the following options:

    Total

    Enter a single amount for the entire date range.

    Quarterly

    Enter separate amounts for each quarter in the date range.

    If you have already entered estimated costs and then change the Time Granularity setting, some data may be lost.

    When you change the setting from Quarterly to Total, all existing quarterly amounts are combined into a single total. The individual quarterly values are permanently removed and cannot be recovered, even if you later change the Time Granularity setting back to Quarterly.

  5. Click Save to apply your changes.

A notification is displayed to confirm that your changes have been applied, and your changes take effect immediately.
