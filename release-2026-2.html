What's new: 2026.2

Discover all the new Uptempo functionality introduced with the 2026.2 release, available from February 25, 2026.

Features marked with the tags Preview and Beta are not activated by default, but can be turned on in your Uptempo instance on request.

For more information, see Uptempo's feature release lifecycle.

Uptempo Platform: Administration

Restrict access to your Uptempo instance based on IP address

You can now configure your Uptempo instance to only allow users with approved IP addresses to sign in.

  • Restrict access to secure networks: Specify exactly which IP addresses are allowed to access your Uptempo instance, ensuring that only users on your corporate VPN or other approved networks can reach sensitive data.

  • Mitigate the risk of credential leaks: Prevent unauthorized access even in the event of a password leak, as login attempts from unrecognized networks are automatically blocked.

  • Flexible configuration: Configure allowed IPs usign individual IPv4 or IPv6 addresses, or broad CIDR ranges, providing the flexibility to support complex infrastructures.

→ Learn more about implementing IP-based access restrictions in your Uptempo instance

Campaign Management: Activities

Improved visibility for in-market date errors

To help ensure that in-market dates are set correctly on all activities, Uptempo now shows a warning when a child activity's in-market dates fall outside the parent activity's date range.

  • Highly visible warnings: Warnings are displayed in the activity setup assistant, in the activity details panel, and on activity bars on the Timeline, making it easy to identify activities that have incorrectly set in-market dates.

  • Improved data consistency: Whenever users set in-market dates, immediate feedback helps them to correct errors right away, ensuring accurate and consistent data entry.

Support for saving Timeline Scale setting in views

When you save your Timeline display settings as a view, your current Timeline Scale setting is now also saved to the view.

Whenever you select a view again later, the saved Timeline Scale (for example, Days, Weeks, Months, etc.) is now also restored alongside all other saved display settings, ensuring a more consistent experience.

→ Learn more about saving Timeline display settings as views

Financial Management: Budgets

Copy activity connections in rollovers

The rollover process now supports copying activity connections on investments.

This new option allows you to keep existing connections between investments and activities from the current fiscal year when you roll over to a new fiscal year. This means that you don't need to manually reconnect investments in your new fiscal year hierarchy to activities, saving time and effort.

Financial Management: Investments

Investment Persistent ID now displayed by default

The Persistent ID for investment line items is now easier to find.

Persistent IDs of investment line items are important during the rollover and monthly reconciliation process. Previously, it was easy to overlook the Persistent ID, as it was hidden by default, and required a click to be displayed. Now, whenever you open an investment's details panel, the Persistent ID is visible right away, making it easier for users to find this key data.
