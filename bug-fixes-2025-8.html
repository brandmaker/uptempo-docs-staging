Bug fixes: 2025.8

To ensure that our users can benefit from bug fixes as soon as possible, we release bug fix updates to the Uptempo platform frequently.

The following bug fixes have been released over the past four weeks, between July 28 and August 22, 2025:

Campaign Management

  • When an attribute option on a Multi-Select List attribute is deactivated:

  • It is no longer shown as an available option when selecting attribute options (if it is not currently selected).

  • It can now be removed as a selected option on attributes where it was selected before being deactivated.

  • It will no longer accept inputs in the percentage field if Percentage Allocation is also enabled on the attribute.

  • The Summary display mode no longer experiences a performance degradation in cases where there are a large number of activities that have Multi-Select List attributes with Percentage Allocation enabled.

  • On the Timeline, after you use Group by on an attribute that results in some activities being in the group "No value set", using the quick search function no longer causes an error.

  • On activities with two distinct attribute dependency chains, editing both of the dependency chains at the same time but then canceling one of the changes no longer causes the action buttons to disappear from both dependency chains.

  • When you rename a view to a name that already has one or more copies (for example renaming a view to "Example View" when "Example View (copy)", "Example View (copy 2)", etc. already exist), the system will no longer unnecessarily auto-append "(copy X )" to the name in cases where the specified name is available (for example if the original "Example View" was deleted or renamed).

  • In the View options menu, a particular sequence of actions no longer causes the system to incorrectly display the "Save as new" and "Save" buttons as deactivated when they should be active.

  • The system now handles switching between Campaign Management and Financial Management better when a view is applied to the Campaign Management Timeline or Summary. If you click on a link to a connected activity from an investment on the Financial Management side, and a view is still applied on the Campaign Management side that does not display that activity, Campaign Management now automatically switches back to the "All activities" view to ensure the activity can be displayed.

  • Connecting an investment line item an activity no longer causes that line item to be displayed four times in the activity's details panel.

  • The investment picker (" Select investment to connect to... ") dialog now has a fixed size to improve usability, and avoid having to scroll past the bottom of the screen.

Financial Management

  • To ensure that Campaign Management access controls are fully applied when a user connects an activity to an investment, the system now also evaluates all attributes attached to the activity against the user's applicable access control policies.
