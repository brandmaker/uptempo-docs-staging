Bug fixes: 2025.10

To ensure that our users can benefit from bug fixes as soon as possible, we release bug fix updates to the Uptempo platform frequently.

The following bug fixes have been released over the past four weeks, between September 22 and October 24, 2025:

API

  • When saving Impact Actual data to a child activity of an activity that already has this type of data, the API no longer improperly returns a 500 Internal Server Error. Instead, it now correctly returns a 400 Bad Request error that specifies that the data can't be saved because the parent activity already contains Impact Actual data.

  • When creating line items, the API no longer takes much more time to complete the request than expected (resulting in a timeout response in some cases).

Campaign Management

  • In attribute dependencies, changing the value of the controlling attribute will no longer cause an error to occur in cases where the dependent attribute already has a value that isn't valid for the new controlling attribute value.

  • When an activity's details panel is open, the Open Comments button no longer covers the Log out button in the profile menu, preventing it from being clicked.

  • When an activity's details panel is open, the Open Comments button no longer stays fixed in place when scrolling the details panel, but instead scrolls with other content.

  • When the Comments panel for an activity is open, it no longer covers the tooltips of the buttons in the header bar, making them unreadable.

Financial Management

  • In the Funding Panel for allocating investment amounts to connected activities, the validation mechanism that checks for overallocated columns now only takes into account columns visible in the current view, preventing validation errors being shown for hidden columns (which can't be fixed from the Funding Panel).

  • To avoid losing unsaved changes, the Funding Panel now prevents the selected view from being changed until all pending changes have been saved.

  • Trying to create a line item for locked cells no longer causes an error.
