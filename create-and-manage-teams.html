Manage access control with teams

The features described on this page are available in Uptempo 2.0 instances. For instructions on how to use this feature in Uptempo 1.0 instances, please refer to this guide.

If you're not sure which version of Uptempo you're using, or if you're interested in upgrading, please get in touch with your Customer Success contact.

To apply activity access controls to your Campaign Management users, you can create teams to manage multiple users as a single entity. When you assign a team to an access control policy, the policy's access permissions automatically extend to all users in the team.

Using teams to apply access controls helps you to save time and manual effort, reduces the risk of configuration errors, and makes ongoing maintenance easier. Whenever users join, leave, or change roles, you can adjust access by simply adding users to or removing them from teams, instead of modifying multiple access control policies.

Create new teams

By default, Campaign Management does not contain any teams. Before you can begin using teams to manage access control permissions, you must first create the teams you want to use. You can create as many teams as you need.

Teams in Uptempo don't need to mirror your org chart.

For access control purposes, it is often more effective to group users by job function or specific tasks rather than by department. This allows you to manage permissions for a specific set of needs as a single, streamlined entity.

Add a new team

You can add a new team to Campaign Management at any time.

  1. In the Activities section, click Settings.

  2. In the Activity Configuration menu, click Access Control > Teams.

  3. On the Teams page, click + Add Team. The New Team dialog opens.

  4. Enter a name for the team in the Name field.

  5. To finish creating the new team, click Create team.

The New Team dialog closes, and the newly created team is displayed in the list of teams.

    You can now add users to the new team. Newly created teams are also immediately available for linking to access control statements.

    Add or remove users in a team

    Manage the members of a team to quickly grant or revoke access control permissions.

    Add users to a team

    You can add users to a team at any time to grant them the permissions defined by the team's linked access control policies.

    Users can be added to multiple teams. If a user is a member of multiple teams, they receive the combined permission scope of all policies linked to all of their teams.

    1. In the Activities section, click Settings.

    2. In the Activity Configuration menu, click Access Control > Teams.

    3. On the Teams page, click on the team to which you want to add users. The team's details panel opens to the Members tab, which shows the team's current members.

    4. On the Members tab, click Add members. The Add new members dialog opens.

    5. In the Add new members dialog, click on a user in the list to select them:

      • Selected users are highlighted with a colored background.

      • You can select multiple users to add. The number of selected users is shown on the Add selected button.

    6. Click Add selected to add the selected users to the team. The selected users are displayed on the Members tab.

    7. To finish adding users, click Save changes.

    The selected users are now members of the team and immediately receive the permissions linked to the team.

      Remove users from a team

      You can remove users from a team at any time to revoke the permissions that are granted by membership in the team.

      1. In the Activities section, click Settings.

      2. In the Activity Configuration menu, click Access Control > Teams.

      3. On the Teams page, click on the team from which you want to remove users. The team's details panel opens to the Members tab, which shows the team's current members.

      4. To remove a user from the team, click Remove member beside that user. The user is removed from the list on the Members tab.

      5. Optional: Repeat step 4 to remove additional users.

      6. To finish removing users, click Save changes.

      The users are removed from the team, and the system immediately revokes all permissions linked to the team from the removed users.

        Edit or remove teams

        You can change any existing team's name, or delete a team if you no longer need it.

        Rename a team

        You can change any team's name at any time.

        1. In the Activities section, click Settings.

        2. In the Activity Configuration menu, click Access Control > Teams.

        3. On the Teams page, click on the team which you want to rename. The team's details panel opens.

        4. Enter a new name for the team into the Name field.

        5. To apply the change, click Save changes.

        The team's name change takes effect immediately.

          Delete a team

          You can delete any team at any time, including teams that still contain users and are still linked to access control policies.

          When you delete a team that contains users, any remaining users are automatically removed from the team (the user accounts are not deleted).

          If a deleted team was still linked to any policies, the removed users also immediately lose the access permissions they had been granted by their membership in the team.

          1. In the Activities section, click Settings.

          2. In the Activity Configuration menu, click Access Control > Teams.

          3. On the Teams page, click on the team which you want to delete. The team's details panel opens.

          4. In the details panel, click Delete team.

          5. In the confirmation dialog, click Yes to finish deleting the team.

          The team is deleted, and removed from the list of teams. Any remaining users are automatically removed from the team. The team is also removed from all policies to which it was linked.

          You can also delete a team by clicking Delete team directly on the team's entry on the Teams page.
