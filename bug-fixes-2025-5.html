Bug fixes: 2025.5

To ensure that our users can benefit from bug fixes as soon as possible, we release bug fix updates to the Uptempo platform frequently.

The following bug fixes have been released over the past four weeks, between April 28 and May 23, 2025:

Administration

User List

  • When new users are invited to Uptempo Financial Management, they no longer fail to appear in the User List in some cases.

Permissions

  • In the Planner subsection of the Permissions page, the Users count for each permission no longer incorrectly shows "0" when there are users assigned to the permission.

API

Authentication

  • In the API documentation, the example request header for authentication no longer incorrectly says "token" instead of "Bearer" (which would prevent the request from working if the example request was used as provided).

Campaign Management

Activities

  • After you create a new activity, the Timeline display mode now ensures that the newly created activity is visible by automatically scrolling to it (and expanding parent activities above it if needed).

  • When creating a new activity using the Add Under button, the activity setup assistant no longer exhibits erroneous behaviour in some cases (such as failing to load, or displaying blank dropdown menus).

  • When you are configuring a multi-select attribute where percentage allocations are enabled, highlighting the percentage value for a selected list option no longer causes that list option to become deselected.

  • When no activity types have been defined in an Uptempo instance yet, the Create Activity button is no longer hidden in the Activities section. Instead, the button is now visible but deactivated, and displays a tooltip to more clearly explain that creating activities is not possible until at least one activity type has been set up.

  • Overriding roll-up values on the Details > Impact tab no longer causes the activity's In-Market Dates to be changed inadvertently.

Activity Configuration

  • For access control statements that contain a condition which specifies activity type group IDs, opening the Conditions info box for the statement no longer incorrectly displays the names of activity types that happen to have the same IDs, but instead correctly displays the names of the specified type groups.

  • For access control statements that contain an IS ONE OF condition and a large number of specified values, opening the Conditions info box for the statement no longer causes the long list of condition values to be cut off.

  • When you try to delete an activity type that still has associated activities, activity rules, or outbound actions, the system now displays a more helpful error message to explain why you can't delete the activity type.

  • The Set your default average deal size field on the Measures > Impact Modeller configuration page now uses the currency set as the Primary Plan Currency (on the Activities > Currencies page), instead of always displaying USD.

Financial Management

Investments

  • The Activity Funding Panel no longer displays duplicate allocation amounts.

  • When a line item's amount is entered in a local currency and converted, trying to allocate the full amount of the line item in the Activity Funding Panel no longer causes an error when trying to save the allocations.

  • The icons used to expand UI items are no longer inconsistent.

  • The Select activity to connect to... dialog no longer fails to display an activity's type under the Type column.

  • Moving an investment under a different parent category or sub-category no longer causes the color-coding and icons that symbolize activity connections to disappear from the investments table.

  • Creating a calculated column when there are already 15 columns or more in the investments table no longer causes an error that prevents the calculated column from being edited or deleted.

Work

All Modules

  • When you stay on the same page for a long time (more than 30 minutes), reloading the page or navigating to a different page no longer causes the navigation sidebar menu to disappear.
