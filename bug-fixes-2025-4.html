Bug Fixes: 2025.4

To ensure that our users can benefit from bug fixes as soon as possible, we release bug fix updates to the Uptempo platform frequently.

The following bug fixes have been released over the past four weeks, during the period of March 24 to April 25, 2025:

Administration

User List

  • The user management system no longer creates a duplicate user if a user is created with the same email as an existing user, but different capitalization (e.g. user.name@email.com and USER.NAME@email.com).

Analytics

Insights

  • The list of dashboards no longer displays a scrollbar when there are more than four dashboards to display.

Campaign Management

API

  • The JSON generated by the API in binary file output is no longer broken.

  • For any GET endpoint that returns the details of a line item, the property hasManuallyAllocatedInvestment no longer returns the value null when no value is set, and instead returns the more accurate value false.

Activities

  • When you open the details panel for any activity in the Timeline display mode, the panel no longer briefly displays a message that the activity is no longer available while the activity's details are loading.

  • In the Timeline display mode, the Zoom Out button is now deactivated when the maximum zoom level is reached, to more clearly indicate when there are no additional zoom levels available.

  • The activity details panel no longer displays the error message that the selected parent is not valid under the Parent field.

  • In the activity details panel, it is no longer possible to enter dates more than 5 years before or after the current date in the date fields on the KPIs and Estimated Costs tabs (previously, doing so would cause an error).

  • When creating an activity, the Next button is no longer incorrectly deactivated for certain combinations of values in the Activity Type and Parent fields.

  • When creating an activity, it is no longer possible to select an activity type that does not have any activity rules defined (previously, doing so would cause an erroneous error message to be shown).

  • When creating an activity, it is no longer possible to select an activity type that has been disabled (previously, doing so would cause activity creation to fail).

  • When creating an activity, it is no longer possible to enter an activity name that is longer than the system limit of 128 characters (previously, doing so would cause activity creation to fail).

  • When creating an activity, the system no longer incorrectly displays an error message on the Budget step when you connect an investment item and then disconnect it again.

  • After creating an activity, the success notification message no longer immediately disappears, and is instead displayed long enough to be read.

  • When you open the details panel for an activity that has already been deleted (e.g. deleted by another user before you had refreshed the page), the system no longer displays an empty details panel, and instead displays an error message.

Activity Configuration

  • When you attempt to delete an activity rule that has already been deleted (e.g. deleted by another user before you had refreshed the page), the system no longer displays an inaccurate error message.

  • The error message that is displayed when you attempt to delete an activity rule that is still in use has been improved to be clearer and more informative.

  • When you set a custom label for Estimated Costs, the wording "estimated costs" is no longer used in the success notification after you make changes to an activity's estimated costs, and the specified custom label is used instead.

Financial Management

Investments

  • In the Activity Funding Panel, you no longer need to press the Delete or Backspace buttons twice after highlighting a column value to remove it.

  • In the Activity Funding Panel, the Activity column is no longer displayed with an extremely wide width when the column contains an activity with a very long name.
