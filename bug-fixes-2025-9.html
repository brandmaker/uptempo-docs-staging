Bug fixes: 2025.9

To ensure that our users can benefit from bug fixes as soon as possible, we release bug fix updates to the Uptempo platform frequently.

The following bug fixes have been released over the past four weeks, between August 25 and September 19, 2025:

API

  • When retrieving activities from the API, using the filtering function no longer causes a timeout error to occur.

  • When an API user resets their Uptempo account password, this no longer causes them to be unable to authenticate into the API.

Campaign Management

  • Updating an attribute dependency no longer causes an error to occur when a list-type attribute in the dependency has deactivated list options.

  • It is no longer possible to create duplicate activity rules (which prevents further rules from being created).

  • Activity rules that have been deactivated are now ignored when checking if they would conflict with adding an attribute to an activity type.

  • The inherited attributes section of the activity details panel no longer shows blank lines for attributes that have been deleted.

  • When a view is selected on the Timeline that includes a selected Color By option, changing the Color By setting to a different option and reloading the page no longer causes the Color By setting to revert to the option saved in the view.

  • The color picker for setting Timeline Visuals (Color By) colors no longer includes the "white" color preset as an available option, as activity bars set to this color are not clearly visible on the Timeline.

  • In the Filter by menu, the arrow on the Select Attribute menu is no longer unresponsive to clicking, but instead opens the menu as expected.

  • In the View Options menu, the Save button is now inactive when no view name has been entered, to prevent the error that occurs when trying to save a view with no name specified.

Financial Management

  • For line items connected to activities where Auto Allocation is active, the system no longer fails to save changes to column amounts in the Funding Panel (despite indicating that changes have been saved).

  • After deleting one or more line items with values in at least one currency column, taking a snapshot of the currency column no longer causes an error to occur.

  • The Transfer Requests page should now take less time to load.

  • Editing an import schedule at the same time as another user no longer causes an error to occur when saving changes.
