Bug fixes: 2025.7

To ensure that our users can benefit from bug fixes as soon as possible, we release bug fix updates to the Uptempo platform frequently.

The following bug fixes have been released over the past four weeks, between June 23 and July 25, 2025:

All Modules

Navigation

  • The sidebar navigation menu now displays animated "placeholders" while it is loading menu items to signal that it is actively loading.

Administration

Look & Feel

  • When you customize the system colors in Look & Feel > Desktop Skinning, the Highlight color setting no longer excludes the Expand and Collapse buttons in the navigation sidebar (which would always default to white).

API

Campaign Management API

  • Fetching all activity types with the API in an instance where one attribute is set at the controlling attribute for more than one dependent attribute no longer causes an error to be returned instead of the requested activity types.

Campaign Management

Activity Configuration

  • When an access control statement gives a user access to an activity through a condition based on attributes, the system no longer denies access when the user tries to modify the attributes on that activity.

  • Deleting an option from an attribute of the type Multi-Select List no longer causes an error to occur.

  • For attribute dependencies where two or more controlling attributes each have more than one dependent attribute, setting the value for a dependent attribute no longer causes an error to occur.

  • For attribute dependency chains, the attributes in the chain no longer fail to appear in the activity setup assistant when the first controlling attribute in the chain is not enabled on the activity type being created.

Activities

  • It is no longer possible to clear the values on required attributes of the type Drop-Down List or Multi-Select List, which would cause the details panel to enter a state in which attribute values couldn't be saved.

  • The list of selected filter options is no longer cut off when a large number of filter options are selected in the Filter by menu.

  • Selecting exactly 36 options on a Multi-Select List attribute with percentage allocation enabled no longer causes the details panel to enter a state in which attribute values couldn't be saved.

  • Selecting a controlling attribute value and then clearing it no longer causes an error to occur when you create a new activity of a type that has an attribute dependency chain.

  • For a dependent attribute that is a Multi-Select List attribute where percentage allocation is enabled, selecting a different controlling attribute option no longer causes the auto-calculated percentage allocations to be incorrect when a selected dependent attribute value is removed (because it is no longer valid for the new controlling attribute option).

  • In the Timeline display mode, an activity's timeline bar is no longer misaligned until the screen is refreshed when you change the activity's start date in the details panel.

  • When using the Add under function to create a new activity, the activity setup assistant no longer displays a blank window when go back to the Details step after reaching the KPI step.

  • When the Impact Modeller feature is not enabled, navigating to the Impact > Actual tab on an activity no longer causes an error to occur. Instead, the Impact section is now hidden.

  • Moving an activity under a different parent no longer causes an error to occur when the user does not have permission to access the new parent's ancestor activities.

  • Opening the Budget tab section on an activity no longer causes an error to occur when the user does not have permission to access the activity's parent or ancestor activities.

Financial Management

Investments

  • In the Activity Funding Panel, the contents of the last column is no longer cut off when the last column is a calculated column.

  • Selecting the Show Unallocated Amounts option in the Filter by Activity Funding menu no longer causes an error to occur when a line item has a column value but no allocation amounts.

Work

Home

  • Opening the board settings for certain boards in the Home module no longer displays an error message, but instead displays the Update board dialog as expected.
